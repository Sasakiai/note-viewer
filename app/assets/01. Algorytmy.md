## Specyfikacja algorytmu

Specyfikacja algorytmu mówi dokładnie, co algorytm ma zrobić. Składa się z:
- nazwy algorytmu i listy argumentów
- warunku początkowego, czyli opisu poprawnych danych wejściowych
- warunku końcowego, czyli opisu poprawnego wyniku

Przykład:
`sum(A,n)`

Wejście:
- `A` - tablica liczb całkowitych
- `n` - liczba naturalna oznaczająca długość tablicy

Wyjście:
suma pierwszych `n` elementów tablicy `A`

## Warunek początkowy

Warunek początkowy określa, jakie dane wejściowe są poprawne.

Przykład: `A` jest tablicą liczb całkowitych, a `n >= 0`.

## Warunek końcowy

Warunek końcowy określa, jaki wynik ma zostać zwrócony dla poprawnych danych wejściowych.

Przykład: algorytm ma zwrócić sumę wszystkich elementów tablicy

## Własność stopu

Algorytm ma własność stopu, jeżeli dla każdych poprawnych danych wejściowych kończy działanie po skończonej liczbie kroków.

Przy uzasadnianiu własności stopu patrzymy zwykle, czy w każdej iteracji pętli jakaś zmienna zbliża się do końca, np. `i++`, `n--`, `node = node.next`

## Częściowa poprawność

Algorytm jest częściowo poprawny, jeżeli dla każdych poprawnych danych wejściowych, jeżeli algorytm się zatrzyma, to zwrócony wynik jest poprawny.

**Ważne:** częściowa poprawność nie mówi, że algorytm na pewno się zatrzyma

## Całkowita poprawność

Algorytm jest całkowicie poprawny, jeżeli:
1. ma własność stopu,
2. po zatrzymaniu zwraca poprawny wynik

Czyli:
`całkowita poprawność = własność stopu + częściowa poprawność`

## Niezmiennik pętli

Niezmiennik pętli to warunek logiczny, który jeśli jest prawdziwy przed wykonaniem iteracji pętli, to pozostaje prawdziwy po jej wykonaniu.

Używa się go do dowodzenia częściowej poprawności algorytmów z pętlami.

## Jak uzasadniać własność stopu

Najczęstszy schemat jest prosty:
- wskazujemy zmienną sterującą, np. `i`, `n`, `node`
- pokazujemy, że w każdej iteracji zmienia się w stronę końca
- pokazujemy, że nie może zmieniać się w nieskończoność

Przykłady:
- w pętli po tablicy `i++`, a `i` jest ograniczone przez `n`
- w pętli `while (n > 0)` zmniejszamy `n--`
- w liście wykonujemy `node = node.next`, więc po skończonej liczbie kroków dojdziemy do `null`

## Jak myśleć o częściowej poprawności

Częściową poprawność najczęściej uzasadniamy tak:
1. wybieramy niezmiennik pętli
2. pokazujemy, że jest prawdziwy przed pierwszą iteracją
3. pokazujemy, że jedna iteracja go nie psuje
4. pokazujemy, że po zakończeniu pętli z niezmiennika wynika poprawny wynik

Na egzamin zapamiętaj: własność stopu mówi, że algorytm się kończy, a częściowa poprawność mówi, że jeśli się kończy, to wynik jest dobry.

## Wzorzec pseudokodu: suma na liście jednokierunkowej

```
sumList(head) {
	sum = 0
	node = head

	while (node != null) {
		sum += node.value
		node = node.next
	}

	return sum
}
```

Specyfikacja:
- wejście: wskaźnik `head` na pierwszy węzeł listy jednokierunkowej
- wyjście: suma wartości wszystkich węzłów listy

Własność stopu:
- w każdej iteracji przechodzimy do następnego węzła
- lista ma skończoną liczbę węzłów, więc po skończonej liczbie kroków `node == null`
