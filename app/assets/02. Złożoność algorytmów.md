## Operacja dominująca

Operacja dominująca to taka operacja, której liczba wykonań najlepiej opisuje czas działania algorytmu.

Najczęściej jest to:
- porównanie elementów
- przypisanie do tablicy
- sprawdzenie warunku w pętli
- porównanie kluczy

Przykład:

```
for i = 0 to n - 1:
	if A[i] == x:
		return i
```

Operacja dominująca: porównanie `A[i] == x`

## Rozmiar danych

Rozmiar danych to funkcja opisująca wielkość wejścia.

Przykłady:
- dla tablicy: `n = len`
- dla CountSort: dwa parametry, `n = długość ciągu`, `m = maksymalna wartość`
- dla kopca: `n = liczba elementów w kopcu`
- dla drzewa: `n = liczba węzłów`

## Pesymistyczna złożoność czasowa

Oznaczenie: `W(n)`

Jest to liczba operacji dominujących w najgorszym przypadku dla danych rozmiaru `n`.

Przykład: w zwykłym wyszukiwaniu sekwencyjnym najgorszy przypadek jest wtedy, gdy elementu nie ma albo jest na końcu. Wtedy:

`W(n) = Θ(n)`

## Przeciętna złożoność czasowa

Oznaczenie: `A(n)`

Jest to średnia liczba operacji dominujących przy założonym modelu losowości danych.

Przykład: jeśli szukany element może być na każdej pozycji z takim samym prawdopodobieństwem, to przeciętnie znajdziemy go mniej więcej w połowie tablicy, więc nadal:

`A(n) = Θ(n)`

## Złożoność pamięciowa

Oznaczenie: `S(n)`

Opisuje, ile dodatkowej pamięci zużywa algorytm jako funkcję rozmiaru danych.

Przykłady:

`S(n) = O(1)`         - stała pamięć
`S(n) = Θ(n)`         - liniowa pamięć
`S(n,m) = Θ(n+m)` - pamięć zależy od dwóch parametrów
