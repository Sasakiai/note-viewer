## Wyszukiwanie sekwencyjne

Działa na ciągu nieposortowanym i posortowanym.

Pseudokod:

```
find(A, n, key) {
	i = 0
	
	while (i < n) {
		if (A[i] == key) {
			return i
		}
		
		i++
	}
	
	return -1
}
```

Założenia
- dane nie muszą być posortowane
- działa na tablicy i liście
- na liście przechodzimy `node = node.next`

Złożoność:

- `W(n) = Θ(n)`
- `A(n) = Θ(n)`
- `S(n) = O(1)`

Najgorszy przypadek: elementu nie ma albo jest na końcu.

## Wyszukiwanie skokami co k

Zakłada, że ciąg jest posortowany.

Idea:
1. sprawdzamy co `k`-ty element
2. gdy trafimy na element większy od szukanego, cofamy się do ostatniego bloku
3. przeszukujemy liniowo ostatnie maksymalnie `k` elementów

Złożoność:

`Θ(n/k + k)`

Dla `k = √n`:

`Θ(√n)`

Dla stałego `k` nadal jest to rząd liniowy:

`Θ(n)`

Na liście dowiązaniowej traci sens, bo skok do indeksu nie jest stałoczasowy.

## Wyszukiwanie binarne

Zakłada:
- ciąg jest posortowany
- mamy szybki dostęp do środka (tablica w RAM)

Pseudokod:

```
binarySearch(S, len, key) {
	left = 0
	right = len - 1
	
	while (left <= right) {
		mid = (left + right) / 2
		
		if (S[mid] == key) {
			return mid
		}
		
		if (S[mid] > key) {
			right = mid - 1
		} else {
			left = mid + 1
		}
	}
}
```

Złożoność:

- `W(n) = Θ(log n)`
- `A(n) = Θ(log n)`
- `S(n) = O(1)`
