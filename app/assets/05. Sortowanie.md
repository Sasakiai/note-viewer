## Stabilność sortowania

Algorytm sortowania jest stabilny, jeśli zachowuje względną kolejność elementów o tej samej wartości.

Przykład:

- `Wejście:  4, 2a, 3, 2b, 1, 2c`
- `Wynik:    1, 2a, 2b, 2c, 3, 4`

Elementy `2a, 2b, 2c` zostają w tej samej kolejności.

Stabilność jest ważna np. przy sortowaniu rekordów po kilku atrybutach.

## SelectionSort

Idea:
1. znajdź minimum
2. zamień je z pierwszym elementem
3. powtarzaj dla reszty tablicy

Złożoność:

- `W(n) = Θ(n^2)`
- `A(n) = Θ(n^2)`
- `S(n) = O(1)`

## InsertionSort

Idea:
1. utrzymujemy posortowany prefiks tablicy
2. bierzemy kolejny element
3. wstawiamy go w odpowiednie miejsce w prefiksie

Złożoność:

- `W(n) = Θ(n^2)`
- `A(n) = Θ(n^2)`
- `S(n) = O(1)`

## Merge (funkcja)

Funkcja `merge` łączy dwa posortowane ciągi w jeden posortowany ciąg.

Złożoność:

- `W(n) = Θ(n)`
- `A(n) = Θ(n)`
- `S(n) = Θ(n)`

## MergeSort

Idea:
1. podziel ciąg na dwie połowy
2. posortuj rekurencyjnie każdą połowę
3. połącz posortowane połowy funkcją `merge`

Pseudokod:

```
mergeSort(S, len) {
	if (len <= 1) {
		return S
	}
	
	m = len / 2
	
	left = mergeSort(S[0:m], m)
	right = mergeSort(S[m:len], len - m)
	
	return merge(left, m, right, len - m)
}
```

Złożoność:

- `W(n) = Θ(n log n)`
- `A(n) = Θ(n log n)`
- `S(n) = Θ(n)`

## QuickSort

Idea:
1. wybierz pivot
2. funkcja `partition` ustawia elementy mniejsze lub równe po lewej, większe lub równe po prawej
3. rekurencyjnie sortuj obie części

Złożoność:

- `A(n) = Θ(n log n)`
- `W(n) = Θ(n^2)`
- `S(n) = O(log n)`

## CountSort

Założenia:
- sortujemy liczby naturalne,
- znamy maksymalną wartość `m`,
- `n` to długość ciągu.

Pseudokod:

```
countSort(a, l) {
	max = maxValue(a, l)
	l1 = max + 1
	
	counts[l1]
	result[l]
	
	for (i = 0; i < l1; i++) {
		counts[i] = 0
	}
	
	for (i = 0; i < l; i++) {
		counts[a[i]]++
	}
	
	for (i = 1; i < l1; i++) {
		counts[i] += counts[i-1]
	}
	
	for (i = l - 1; i >= 0; i--) {
		result[--counts[a[i]]] = a[i]
	}
}
```

`counts` po pierwszym przejściu:

`counts[x] = liczba wystąpień wartości x`

`counts` po sumowaniu prefiksowym:

`counts[x] = liczba elementów <= x`

Po ostatniej pętli:

`result = posortowany ciąg wejściowy`

Złożoność:

- `W(n, m) = Θ(n + m)`
- `A(n, m) = Θ(n + m)`
- `S(n, m) = Θ(n + m)`

## RadixSort

Idea:
1. sortujemy liczby po kolejnych cyfrach
2. zwykle od najmniej znaczącej cyfry
3. do sortowania po cyfrze używamy stabilnego sortowania, np. CountSort
