#PJATK #ASD 

Stos, kolejka i deque są abstrakcyjnymi strukturami danych.

## Stos

Stos działa według zasady:

`LIFO - Last In, First Out`

Czyli jako pierwszy usuwany jest element dodany ostatnio.

Podstawowe operacje:

```
push(x)
pop()
top()
isEmpty()
```

Przebieg operacji:

```
push:
nowy element trafia na szczyt stosu.

pop:
usuwany jest element ze szczytu stosu, a nowym szczytem staje się element pod nim.

top:
zwracany jest element ze szczytu, ale struktura się nie zmienia
```

Przy dobrej implementacji wszystkie podstawowe operacje mają koszt:

`O(1)`

Stos pasuje do problemów, gdzie interesuje nas ostatnio dodany element, np. cofanie operacji, sprawdzanie nawiasów, obsługa rekurencji.

Efektywne implementacje:
- tablica z indeksem szczytu `top`
- lista jednokierunkowa z dostępem do początku

W obu przypadkach `push`, `pop`, `top` mogą działać w `O(1)`.

## Kolejka

Kolejka działa według zasady:

`FIFO - First In, First Out`

Czyli jako pierwszy usuwany jest element dodany najdawniej.

Podstawowe operacje:

```
enqueue(x)
dequeue()
front()
isEmpty()
```

Przebieg operacji:

```
enqueue:
nowy element trafia na koniec kolejki

dequeue:
usuwany jest element z początku kolejki, czyli ten, który jest tam najdłużej

front:
zwracany jest pierwszy element, ale bez jego usuwania
```

Przy dobrej implementacji z dostępem do początku i końca kolejki wszystkie podstawowe operacje mają koszt:

`O(1)`

Kolejka pasuje do problemów, gdzie trzeba obsłużyć elementy w kolejności przyjścia, np. kolejka klientów, zadania do wykonania, BFS, najdawniej dodane zgłoszenie.

Efektywne implementacje:
- tablica cykliczna z indeksami początku i końca
- lista jednokierunkowa z wskaźnikami `head` i `tail`

W obu przypadkach `enqueue`, `dequeue`, `front` mogą działać w `O(1)`.

## Deque

Deque, czyli kolejka dwustronna, pozwala dodawać i usuwać elementy z obu końców.

Podstawowe operacje:

```
pushFront(x)
pushBack(x)
popFront()
popBack()
front()
back()
```

Przebieg operacji:

```
pushFront:
dodaje element na początek

pushBack:
dodaje element na koniec

popFront:
usuwa i zwraca element z początku

popBack:
usuwa i zwraca element z końca

front:
zwraca element z początku

back:
zwraca element z końca
```

Przy dobrej implementacji, operacje na obu końcach mają koszt:

`O(1)`

Efektywne implementacje:
- tablica cykliczna
- lista dwukierunkowa z dostępem do obu końców

Deque trudniej zaimplementować efektywnie na zwykłej liście jednokierunkowej, bo `popBack` nie jest wtedy stałoczasowe.

## Co zapamiętać o implementacjach

- stos: tablica albo lista jednokierunkowa
- kolejka: tablica cykliczna albo lista z `head` i `tail`
- deque: tablica cykliczna albo lista dwukierunkowa

Na egzaminie zwykle trzeba nie tylko nazwać implementację, ale też powiedzieć, które wskaźniki lub indeksy trzeba przechowywać, żeby operacje miały `O(1)`.
