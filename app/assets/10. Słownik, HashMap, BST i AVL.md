## Słownik

Słownik przechowuje pary:

`klucz -> wartość`

klucze są zwykle unikatowe.

Podstawowe operacje:

```
search(key)
insert(key, value)
delete(key)
```

Słownik pasuje do problemów, gdzie trzeba szybko znaleźć dane po identyfikatorze, np. `clientId`, `productId`, `PESEL`.

## HashMap

HashMap, czyli tablica mieszająca, używa funkcji mieszającej, która zamienia klucz na indeks w tablicy.

Przebieg operacji:

```
search:
obliczamy hash klucza, przechodzimy do odpowiedniego miejsca w tablicy i szukamy pary o tym kluczu

insert:
obliczamy hash klucza i zapisujemy parę klucz-wartość w odpowiednim miejscu

delete:
obliczamy hash klucza, znajdujemy parę o tym kluczu i ją usuwamy.
```

Średnie złożoności przy dobrej funkcji mieszającej:

```
search = O(1)
insert = O(1)
delete = O(1)
```

Pesymistycznie, gdy wiele kluczy trafia w to samo miejsce:

`O(n)`


Kolizja występuje wtedy, gdy różne klucze trafiają do tego samego miejsca w tablicy. Można ją obsługiwać np. przez listy w komórkach tablicy albo przez szukanie innej wolnej pozycji.

HashMap jest dobra do szybkiego dostępu po kluczu, ale nie jest dobra do operacji zależnych od porządku, takich jak minimum, maksimum, następnik, poprzednik, czy wyszukiwanie zakresu.

## BST

BST, czyli Binary Search Tree, to drzewo binarne wyszukiwań.

Własność BST:

klucze w lewym poddrzewie są mniejsze od klucza węzła
klucze w prawym poddrzewie są większe od klucza węzła

Dzięki temu przy wyszukiwaniu nie trzeba przeglądać całego drzewa.

Przebieg operacji `search`:
1. Zaczynamy od korzenia.
2. Jeśli szukany klucz jest równy kluczowi węzła, znaleźliśmy element.
3. Jeśli szukany klucz jest mniejszy, idziemy do lewego poddrzewa.
4. Jeśli jest większy, idziemy do prawego poddrzewa.
5. Jeśli dojdziemy do pustego miejsca, klucza nie ma.

Przebieg operacji `insert`:
1. Schodzimy po drzewie tak jak przy search.
2. Szukamy miejsca, w którym powinno znaleźć się nowy klucz.
3. Nowy węzeł dodajemy jako liść.
4. Własność BST musi pozostać zachowana.

Przebieg operacji `delete`:
1. Najpierw znajdujemy usuwany węzeł.
2. Jeśli węzeł jest liściem, po prostu go odłączamy.
3. Jeśli ma jedno dziecko, podłączamy to dziecko w jego miejsce.
4. Jeśli ma dwoje dzieci, zastępujemy go następnikiem, czyli najmniejszym elementem z prawego poddrzewa, a potem usuwamy następnika z jego starego miejsca.

Złożoność operacji BST zależy od wysokości drzewa `h`, ale dla dobrze ułożonego drzewa:

`O(log n)`

Pesymistycznie :`O(n)`, bo BST może zdegenerować się do listy, np .przy wstawianiu danych już posortowanych.

## AVL

AVL to samorównoważące się BST. Ma tę samą własność porządku co BST, ale dodatkowo pilnuje wysokości poddrzew.

Warunek AVL:

`dla każdego węzła wysokości lewego i prawego poddrzewa różnią się najwyżej o 1`

Często opisuje się to współczynnikiem zrównoważenia:

`bf = wysokość lewego poddrzewa - wysokość prawego poddrzewa`

Dopuszczalne wartości:

`-1, 0, 1`

Jeśli po wstawieniu albo usunięci wartość wyjdzie poza ten zakres, drzewo trzeba naprawić rotacjami.

Przebieg operacji `search` w AVL:

`search` działa tak samo jak w BST.

Przebieg operacji `insert` w AVL:
1. Najpierw wstawiamy nowy węzeł tak jak w zwykłym BST.
2. Następnie wracamy w górę od miejsca wstawienia.
3. Aktualizujemy wysokości i sprawdzamy balans kolejnych węzłów.
4. Jeśli jakiś węzeł jest niezrównoważony, wykonujemy odpowiednią rotację.
5. Po rotacji drzewo nadal spełnia własność BST i odzyskuje warunek AVL.

Przebieg operacji `delete` w AVL:
1. Najpierw usuwamy węzeł tak jak w zwykłym BST.
2. Następnie wracamy w górę od miejsca usunięcia.
3. Aktualizujemy wysokości i sprawdzamy balans.
4. Jeśli pojawi się niezrównoważenie, wykonujemy rotację.
5. Po usunięciu może być potrzebna naprawa na więcej niż jednym poziomie.

Rodzaje przypadków naprawy:

```
LL (left-left):
Problem powstał po lewej stronie lewego dziecka.
Naprawa: jedna rotacja prawa.

RR (right-right):
Problem powstał po prawej stronie prawego dziecka.
Naprawa: jedna rotacja lewa.

LR (left-right):
Problem powstał po prawej stronie lewego dziecka.
Naprawa: najpierw rotacja lewa na lewym dziecku, potem rotacja prawa.

RL (right-left):
Problem powstał po lewej stronie prawego dziecka.
Naprawa: najpierw rotacja prawa na prawym dziecku, potem rotacja lewa.
```

Rotacja jest operacją lokalną. Zmienia kilka połączeń między węzłami, ale zachowuje porządek BST.

Złożoności AVL:

```
search = O(log n)
insert = O(log n)
delete = O(log n)
rotacja = O(1)
```

AVL jest lepsze od zwykłego BST wtedy, gdy potrzebujemy gwarancji logarytmicznego czasu operacji. Kosztem jest większa złożoność implementacji, bo po modyfikacjach trzeba pilnować balansu.
